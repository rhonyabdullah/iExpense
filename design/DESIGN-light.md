## Brand & Style
The brand personality is professional, disciplined, and reliable, designed to instill a sense of financial control and clarity. The target audience includes professionals and organized individuals who require a streamlined, high-efficiency interface for tracking personal capital.

This design system adopts a **Corporate Modern** style, strictly adhering to Material Design 3 (M3) principles. It leverages high-contrast surface treatments and intentional whitespace to reduce cognitive load. The aesthetic is "Functional Elegance"—where every visual cue serves a utility, ensuring the interface feels like a sophisticated tool rather than a decorative app.

## Colors
The palette is rooted in the specified M3 tonal ranges. Primary purple provides a sense of premium authority, while the secondary muted lavender ensures the UI remains approachable. Tertiary rose tones are reserved for specific accents, such as savings goals or highlighted categories.

The system uses **Tonal Palettes** for state management (Hover, Focus, Pressed). In Dark Mode, surfaces use a 12% primary overlay to indicate elevation, while Light Mode utilizes the Neutral-Variant for subtle containment. High contrast is maintained by ensuring all text-on-background pairings meet WCAG AA standards.

## Typography
This design system utilizes **Manrope** for display and headline roles to provide a modern, geometric personality that feels refined and high-tech. **Inter** is used for all body and label roles due to its exceptional legibility at small sizes, which is critical for financial data and line-item auditing.

Large display sizes are reserved for "Total Balance" or "Monthly Summary" views. On mobile devices, `displayLarge` should scale down to `headlineLarge` to prevent horizontal clipping. Line heights are generous to ensure clear separation between rows of financial transactions.

## Layout & Spacing
The system utilizes a **Fluid Grid** model with a base 4dp increment.

- **Mobile:** 4-column grid, 16px margins, 16px gutters.
- **Tablet:** 8-column grid, 24px margins, 24px gutters.
- **Desktop:** 12-column grid, 32px margins, 24px gutters. Content should be capped at a maximum width of 1280px for readability.

Spacing follows the M3 philosophy: tight (4-8px) for related grouping, and loose (24-32px) for section headers and distinct content blocks. Vertical rhythm is maintained by aligning all text baselines to the 4px grid.

## Elevation & Depth
Elevation is communicated through **Tonal Layers** rather than heavy shadows. The system follows the M3 "Surface Container" hierarchy:

1.  **Level 0 (Flat):** Main background (Neutral).
2.  **Level 1 (Elevated):** Content cards and navigation bars (Neutral + 5% Primary tint).
3.  **Level 2 (Raised):** Interactive elements like search bars.
4.  **Level 3 (Modal):** Dialogs and pickers (Neutral + 11% Primary tint).

Shadows are used sparingly, appearing only on Level 2 and above, utilizing a soft, diffused ambient shadow with a subtle Primary tint to maintain color harmony even in the depth model.

## Shapes
The shape language follows a "Structured Softness" approach.

- **Small (8px):** Used for buttons, chips, and text fields.
- **Medium (12px):** Used for transaction cards and small data visualization containers.
- **Large (16px):** Used for main dashboard containers and navigation drawers.
- **Extra-Large (28px):** Used for Floating Action Buttons (FAB) and top-level modal sheets.

This tiered approach ensures that larger structural elements feel significantly more "contained" than the interactive buttons nested within them.

## Components
- **Buttons:** Use the Primary color for high-emphasis "Add Expense" actions. Use "Small" roundedness (8px). Text must be `labelLarge`.
- **Cards:** For transactions, use Level 1 elevation with "Medium" roundedness (12px). Include 16px internal padding for financial clarity.
- **Input Fields:** Use the "Filled" M3 style with a visible bottom stroke. Use the Neutral-Variant for the background to ensure they stand out against Level 0 surfaces.
- **Chips:** For expense categories (e.g., "Food", "Travel"), use "Small" roundedness with a Secondary color outline.
- **Lists:** Transaction lists should use a horizontal 16px padding with 72px fixed-height rows for consistency across devices.
- **Floating Action Button (FAB):** The primary entry point for a new expense. Use "Extra-Large" (28px) roundedness and the Tertiary color for high visual prominence.
- **Data Visualization:** Line and bar charts should use Primary and Tertiary colors for data series, with a stroke weight of 2dp for clear visibility.
